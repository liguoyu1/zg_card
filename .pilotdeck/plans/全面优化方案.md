## 一、🔴 攻击流程与动画系统（最高优先级）

### 1. AI 攻击无动画 — 与玩家攻击流程不一致

**现状对比**:

| 步骤 | 玩家攻击 | AI 攻击 (当前) |
|------|---------|---------------|
| 选攻击者 | 点击己方随从 → 高亮 | `attackers.where(c.canAttack)` 列表化，无视觉效果 |
| 选目标 | 点击敌方 → 嘲讽检查 | `random` 直接选 |
| 攻击动画 | `_triggerAttackAnimation` 500ms | **跳过，无动画** |
| 伤害数字 | ⚠️ 已有组件但未集成 | **跳过** |
| 死亡动画 | ⚠️ 已有组件但未触发 | 卡牌即时移除 |
| 逐次延迟 | 玩家手动操作间隔 | **for 循环瞬间全部执行** |

**定位**: `game_screen.dart` 第 871-899 行 `_aiAttack()` 方法。

**方案**:
```dart
// game_screen.dart - _executeAITurn() 改为 async
Future<void> _executeAITurn() async {
  final notifier = ref.read(aiGameProvider.notifier);
  final state = ref.read(aiGameProvider);
  if (state == null) return;
  final aiId = state.activePlayer.id;

  await Future.delayed(const Duration(milliseconds: 300));
  if (!mounted) return;

  _aiPlayCards();

  await Future.delayed(const Duration(milliseconds: 500));
  if (!mounted) return;

  await _aiAttack();  // ✅ 改为 await

  await Future.delayed(const Duration(milliseconds: 500));
  if (!mounted) return;

  notifier.endTurn(aiId);
  ref.read(gameScreenStateProvider.notifier).startPlayerTurn();
}

// _aiAttack() 改为 async 队列
Future<void> _aiAttack() async {
  final notifier = ref.read(aiGameProvider.notifier);
  final state = ref.read(aiGameProvider);
  if (state == null) return;
  final aiId = state.activePlayer.id;

  final attackers = state.activePlayer.board.where((c) => c.canAttack).toList();

  for (final attacker in attackers) {
    if (!mounted) return;
    final curState = ref.read(aiGameProvider);
    if (curState == null) return;
    final humanPlayer = curState.opponent;

    // ✅ 选目标（与玩家逻辑一致：嘲讽优先）
    String? targetId;
    final tauntMinions = humanPlayer.board.where((c) => c.hasTaunt).toList();
    if (tauntMinions.isNotEmpty) {
      targetId = tauntMinions[Random().nextInt(tauntMinions.length)].id;
    } else if (humanPlayer.board.isNotEmpty) {
      targetId = humanPlayer.board[Random().nextInt(humanPlayer.board.length)].id;
    }

    // ✅ 触发攻击动画 + 伤害数字 + 死亡动画（与玩家完全一致）
    if (targetId != null) {
      _triggerAttackAnimation(attacker.id, targetId);
      notifier.minionAttack(aiId, attacker, targetId);
    } else {
      _triggerAttackAnimation(attacker.id, 'hero');
      notifier.minionAttackHero(aiId, attacker);
    }

    // ✅ 逐次延迟
    await Future.delayed(const Duration(milliseconds: 700));
  }
}
```

### 2. 风怒无限攻击 Bug — 精确根因

**根因**: `game_rules.dart` 第 346-347 行 `TurnService.startTurn` 重置随从状态：
```dart
// ❌ 只重置了 hasAttackedThisTurn 和 isDormant，遗漏了 hasUsedFirstWindfuryAttack
final resetBoard = player.board.map((card) => 
  card.copyWith(hasAttackedThisTurn: false, isDormant: false)
).toList();
```

这导致风怒随从攻击两次后 `hasUsedFirstWindfuryAttack` 永为 true，`canAttack` 短路永远返回 true。

**修改位置**: `game_rules.dart:347`

**修复**:
```dart
final resetBoard = player.board.map((card) => 
  card.copyWith(
    hasAttackedThisTurn: false,
    isDormant: false,
    hasUsedFirstWindfuryAttack: false,  // ✅ 每回合重置风怒状态
  )
).toList();
```

同时修正 `card.dart:89-96` `canAttack` 的更清晰逻辑：
```dart
bool get canAttack {
  if (!isMinion || isDormant) return false;
  if (hasCharge) return true;
  if (hasWindfury) {
    return !hasAttackedThisTurn || !hasUsedFirstWindfuryAttack;
  }
  return !hasAttackedThisTurn;
}
```

### 4. 死亡动画触发 — 幽灵卡牌方案

**现状**: `board_card.dart` 已有 `CardAnimationState.dying` 动画（缩放0 + 旋转 + 淡出 400ms），但从未触发。

**方案**: 在 `_GameScreenState` 中维护幽灵卡牌列表：
```dart
final Set<String> _dyingCardIds = {};
final Map<String, DateTime> _dyingStartTimes = {};

// 检测 health ≤ 0 的卡牌
void _detectDeadMinions(GameState state) {
  for (final card in [...state.player1.board, ...state.player2.board]) {
    if (card.health <= 0 && !_dyingCardIds.contains(card.id)) {
      _dyingCardIds.add(card.id);
      _dyingStartTimes[card.id] = DateTime.now();
      animStates[card.id] = CardAnimationState.dying;
      // 400ms 后清理幽灵
      Future.delayed(const Duration(milliseconds: 450), () {
        if (mounted) {
          _dyingCardIds.remove(card.id);
          // 通知 provider 清除死亡卡牌
          ref.read(aiGameProvider.notifier).removeDeadMinions();
          setState(() {});
        }
      });
    }
  }
}
```

**注意**: 需要在 `GameStateNotifier` 中新增 `removeDeadMinions()` 方法：
```dart
void removeDeadMinions() {
  if (state == null) return;
  state = state!.copyWith(
    player1: state!.player1.copyWith(
      board: state!.player1.board.where((c) => c.health > 0).toList(),
    ),
    player2: state!.player2.copyWith(
      board: state!.player2.board.where((c) => c.health > 0).toList(),
    ),
  );
}
```

void _onHandCardTap(domain.Card card, GameState gameState) {
  if (!_isPlayerTurn) return;
  final player = gameState.getCurrentPlayer(widget.playerId);
  if (card.cost > player.mana) return;

  if (card.isMinion && player.boardCount < 7) {
    ref.read(aiGameProvider.notifier).playCard(widget.playerId, card);

**阵型检测**（`combo_system.dart` 扩展）:
```dart
enum Formation { longSnake, craneWing, squareCircle, spearHead }

Formation? detectFormation(List<BoardSlot> board) {
  final cards = board.where((s) => s.card != null).toList();
  if (cards.length < 3) return null;
  
  final allSameFaction = cards.every(
    (s) => s.card!.owner == cards.first.card!.owner
  );
  final leftRight = cards.length == 2 && cards.first.position == 0 && cards.last.position == 2;
  
  if (allSameFaction && cards.length == 3) return Formation.longSnake;    // 长蛇阵
  if (leftRight && cards.first.card!.owner == cards.last.card!.owner) return Formation.craneWing; // 鹤翼阵
  if (cards.length == 3 && !allSameFaction) return Formation.squareCircle; // 方圆阵
  
  return null;
}
```

**阵型效果**:
| 阵型 | 效果 |
|------|------|
| 长蛇阵 (3同派系) | 全体+1 攻击力 |
| 鹤翼阵 (左右同派系) | 两侧随从获得嘲讽 |
| 方圆阵 (3满位) | 全体+1 生命值 |
| 锥形阵 (左2同派系+右1) | 最左侧随从获得冲锋 |
