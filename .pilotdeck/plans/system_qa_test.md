# 系统性功能QA测试计划

## 目标
构建完整的端到端集成测试套件，覆盖游戏核心玩法全流程、边缘场景和交互逻辑。包含一个可直接运行的 `game_flow_test.dart`，模拟完整对局验证每个状态转换正确性。

---

## 测试套件 `test/integration/game_flow_test.dart`

### 1️⃣ 完整对局流程（核心验证）
| # | 步骤 | 验证点 |
|---|------|--------|
| 1 | `initGame` | 起手4张牌, mana=1, phase=mulligan |
| 2 | `startTurn` | mana恢复满, 抽1张, board重置攻击状态 |
| 3 | `playCard(随从)` | mana消耗正确, board+1, hand-1 |
| 4 | `useHeroPower` | 消耗2mana, 获得2甲 |
| 5 | `playCard(法术)` | mana消耗正确, 效果生效, hand-1, board不变 |
| 6 | `minionAttackHero` | 敌方英雄扣血, 攻击者标记已攻击 |
| 7 | `minionAttack(随从)` | 双方扣血/圣盾抵消/亡语触发 |
| 8 | `endTurn` | activePlayerId切换, turn+1 |
| 9 | AI对手出牌 | 对方手牌-1, board+1 |
| 10 | 回合循环 | mana逐步递增到10, 手牌不超过10 |
| 11 | 胜负判定 | 英雄≤0血 → phase=ended, winnerId正确 |

### 2️⃣ 圣盾交互矩阵（12种组合）
| 攻击方 | 防守方 | 预期结果 |
|--------|--------|---------|
| 普通 | 普通 | 双方扣血 |
| 圣盾 | 普通 | 防守方扣血, 攻击方圣盾抵消 |
| 普通 | 圣盾 | 攻击方扣血, 防守方圣盾移除 |
| 圣盾 | 圣盾 | 双方圣盾移除, 均不扣血 |
| 剧毒 | 普通 | 目标直接死亡 |
| 剧毒 | 圣盾 | 圣盾抵消剧毒, 目标存活 |
| 圣盾+风怒 | 普通 | 2次攻击机会 |

### 3️⃣ 亡语触发链
| 场景 | 预期 |
|------|------|
| 随从被杀 → 亡语 | 召唤1/1民兵到战场 |
| 法术消灭 → 亡语 | 同样触发 |
| 圣盾抵消 → 亡语 | 不触发（未死亡） |
| 连续死亡 → 连环亡语 | 依次触发 |

### 4️⃣ AI行为验证
| 场景 | each难度预期 |
|------|-------------|
| 手牌满费 | simple随机, normal随从优先, hard战吼优先, abyss最优 |
| 空场出牌 | 出随从(所有难度) |
| 攻击目标选择 | simple随机, normal嘲讽→低血, hard有利交换, abyss预判威胁 |
| combo激活条件 | simple否, normal随机, hard手牌≥5, abyss手牌≥3 |

### 5️⃣ 边界条件
| # | 场景 | 预期 |
|---|------|------|
| 1 | 满手牌(10张)抽牌 | 抽到的牌消失(不爆牌) |
| 2 | 满场(7个)出随从 | 操作被拒绝, state不变 |
| 3 | 0法力出牌 | canPlayCard返回false |
| 4 | 牌库空抽牌 | 疲劳伤害递增(1,2,3...) |
| 5 | 双方同时死亡 | 平局, winnerId=null |
| 6 | 法术targetId=null | 默认打英雄 |
| 7 | 变形+亡语 | 变形后不触发原亡语 |
| 8 | 沉默圣盾 | 圣盾移除, 再次受伤正常扣血 |

### 6️⃣ 组合技能验证
| 组合 | 触发条件 | 效果验证 |
|------|---------|---------|
| 合纵连横 | 手牌同时有Z008+Z009 | 所有随从+1/+1 |
| 机关术 | M008+M009 | 召唤2个3/3 |
| 战国双璧 | B010+B009 | 所有随从冲锋 |
| 儒道双修 | R008+D008 | 抽3回4 |


- 全部160+断言通过
- 0个预期外状态变化
- 完整回合模拟到一方死亡结束
