# 战国风炉石传说 - 功能开发计划 (GDD-FUNC)

## 项目概述
- **类型**: 卡牌对战游戏 (类炉石传说)
- **核心玩法**: 1v1回合制卡牌对战
- **技术栈**: Flutter + Flame
- **当前阶段**: Phase 0 - 骨架搭建

---

## 架构设计原则

### 代码质量要求
- [ ] **解耦**: 核心逻辑与UI分离，事件驱动通信
- [ ] **规范**: Dart规范 + flutter_lints严格模式
- [ ] **清晰**: 单一职责，方法不超过50行
- [ ] **无冗余**: DRY原则，公共逻辑提取到util
- [ ] **预留接口**: 美术/音效/动画全部预留接口和占位

### 美术资源预留
- [ ] 资源路径常量定义
- [ ] 占位组件接口
- [ ] 动画回调接口
- [ ] 音效触发接口

### 架构分层
```
lib/
├── domain/           # 领域层 - 纯逻辑，无依赖
│   ├── models/       # 数据模型
│   ├── events/      # 游戏事件(出牌/攻击/技能)
│   ├── services/    # 核心服务(战场/效果/AI)
│   └── rules/       # 游戏规则
├── data/             # 数据层 - 配置与持久化
│   ├── cards/       # 卡牌数据
│   ├── heroes/      # 英雄数据
│   └── repository/  # 存档仓库
├── presentation/     # 表现层 - Flutter UI
│   ├── screens/    # 页面
│   ├── widgets/    # 组件
│   └── providers/  # Riverpod状态
└── core/            # 核心工具
    ├── utils/      # 工具函数
    └── constants/  # 常量
```

### 设计模式
- [ ] **事件驱动**: `GameEvent` → `EventBus` → `Handler`
- [ ] **策略模式**: 卡牌效果抽象为 `CardEffect` 接口
- [ ] **状态模式**: `GameState` 不可变快照
- [ ] **观察者模式**: UI订阅状态变化

### 状态管理
- [ ] Riverpod Provider 管理游戏状态
- [ ] `AsyncNotifier` 处理异步(存档/加载)
- [ ] `StateNotifier` 管理UI状态
